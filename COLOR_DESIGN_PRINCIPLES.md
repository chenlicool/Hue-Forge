# 色彩系统 v2.0 设计原理与核心思路

基于对 Figma 设计稿的分析，Figma 中的色板采用了一套非常严谨且现代的**基于 HSB 模型 + WCAG 对比度校验**的调色思路。本系统旨在生成专业、无障碍且视觉协调的 UI 渐变色阶。

## 1. 核心模型：基于二维坐标系的 HSB (S-B) 曲线拉伸
这个设计工具并不是凭感觉“挑选”颜色，而是在**饱和度 (Saturation)** 和 **亮度 (Brightness)** 的二维直角坐标系中，通过绘制一条曲线来一次性生成一整套渐变色阶（10-100 共10个层级）。
- **X轴（饱和度 S）：** 0% -> 100%
- **Y轴（亮度 B）：** 0% -> 100%
- **原理：** 对于任意给定的**色相 (Hue)**，通过调整这条曲线上各个关键节点的位置，可以平滑地决定深色到浅色的饱和度和亮度变化轨迹，保证生成的渐变色自然流畅，避免出现“脏色”。

## 2. 标准化的亮度递减阶梯（Lightness/Brightness Scale）
在 10 到 100 的 10 个色阶中，设计系统强制规定了一条**恒定的亮度骨架 (B值)**。从数据中提取出了一个绝对固定的 B 值递减数列：
- 10: `B: 95`
- 20: `B: 87`
- 30: `B: 78`
- 40: `B: 68`
- 50: `B: 56`
- 60: `B: 45`
- 70: `B: 35`
- 80: `B: 28`
- 90: `B: 21`
- 100: `B: 15`

**作用：** 这些设定的 B 值控制了明暗节奏。无论当前生成的是紫罗兰（如 `#5237D5`，H250）还是其他颜色，都拥有同样节奏的明暗降低速度。这使得最终生成的不同主色调，如果在同一色阶（例如同为 50 色阶），它们在视觉上的“重量”和“明暗感”是完全一致的。

## 3. 深切无障碍设计的 WCAG 对比度
每个颜色块下方，系统都会计算并体现准确的“对比度数值”（如 `18.68 : 1`, `4.75 : 1`）以及**WCAG 无障碍评级（AAA 或 AA）**。
- **浅色域 (10-40)：** 必须达到 **AAA** 级（通常是深色文字在浅色背景上）。
- **中间色调 (50-60)：** 在保证作为品牌主色足够亮眼的同时，满足 **AA** 级标准。
- **深色域 (70-100)：** 再次回归到 **AAA** 级（通常是白色文字在深色背景上）。

这表明该色板生成系统不仅追求视觉的“好看”，同时极其看重在实际 UI 落地时文字是否清晰可见，保证了高 Accessibility（无障碍）。

## 4. 针对不同颜色的独立校准 (Per-Hue Adjustment)
系统支持针对不同主色调进行独立配置。例如设计稿中存在的：
- `250/70`（蓝紫色系，如 `#5237D5`）
- `100/40`
- `3/50`

**原因：** 不同的颜色（Hue）本身带有不同的感知明度。例如纯黄色天生就比纯蓝色亮得多。因此，需要允许不同的色相拥有属于自己的独立 S-B 曲线，从而修正这种人眼感知的明度偏差。

## 总结与工作流模型
这套色彩系统类似于目前最顶尖的 UI 库底层生成逻辑，其工作流程可总结为：
1. 确定主色相（Hue）。
2. 在坐标系里画一条平滑曲线，定义一系列的 Saturation 和 Brightness 组合。
3. 固定十个阶梯（10-100）的关键 B 值和对应的 S 值。
4. 系统利用图表中的锚点算出具体的 S 和 B，结合设定的 H 生成 Hex 色值。
5. 实时校验每一阶颜色相对于黑白文本的 WCAG 比例，以确保无障碍设计。

## 一、 需求复述与拆解

**先定骨架（灰阶/明度阶梯）：**
- **核心：** 无需手动画曲线，而是先定义整个色板的层级数量（例如 10 级或 11 级）。
- **规则约束：** 系统自动计算并保证这些阶梯满足 W3C 对比度。强制以中线为界：前半段（浅色）保证与黑色文字搭配通过对比度测试；后半段（深色）保证与白色文字搭配通过对比度测试。

**定义基因（色环与色相选色）：**
- **核心：** 基于色相环（Hue）来批量生成主色系。
- **规则约束：** 提供快捷预设（平分 6 色、9 色、12 色、24 色）。支持选中某个具体色相进行角度（Hue 值）微调。
- **智能化：** 根据计算出的色相角度（0-360°），自动匹配对应的常规颜色名称（如 Red, Orange, Amber 等）。

**全局矩阵阵列与智能辅助曲线：**
- **核心：** 结合前两步，展示一个交叉的色板矩阵（多行 Hue × 多列灰阶）。
- **视觉反馈：** 展示每个色块的具体梯度，并直观提示是否满足 W3C 对比度标准。
- **辅助工具：** 侧边展示一条“智能建议曲线”，不是用来瞎拖拽的，而是指示为了满足对比度标准，参数应该落在哪个区间。

**末端微调能力：**
- **核心：** 在色阶矩阵下方提供参数输入口。
- **联动机制：** 可以单独修改某一个层级的 H（继承自色环角度）和 B 甚至 S 数值。修改后瞬间反馈，显示新的颜色并更新对比度校验结果。

**生态接入：一键到 Figma：**
- **核心：** 顶部提供一个导出入口，点击复制为特定的 SVG 代码。
- **结果：** 直接 Ctrl+V 粘贴进 Figma，即可得到排列整齐、带有色值标签的矢量图形色板。

## 二、 解决方案与产品设计图景

基于你的构想，这个工具已经从“绘图板”升级成了**“色彩编译器”**。为了支撑这种高密度的状态计算和实时反馈，我推荐这套工具的前端架构采用 React (Vite 搭建) 来管理复杂的参数状态，并配合极具科幻感和专业感的 Dark Mode（暗黑风格） UI 进行表现。

以下是我的整体设计方案：

### 1. 界面布局层 (UI Layout)
界面我们将划分为顶栏、左侧控制台、中央舞台和右侧洞察面板四个区域，营造一种“驾驶舱”的高级感。

- **顶栏 (Top Bar)：**
  - 工具名称与当前配置的简述。
  - 醒目的 「Copy SVG for Figma」 按钮（带微动画反馈）。
- **左侧控制台 (The DNA Console)：**
  - **阶梯控制器 (Steps Engine)：** 输入框/滑块来设定层级数量（比如 10）。下方直观展示一条纯灰色的基础明度骨架，并直接将黑白文字放上去演示。
  - **色相选择盘 (Hue Wheel Generator)：** 一个基于 HSL 环形设计的精美控件。中间放预设按钮（6 | 9 | 12 | 24），环上出现对应数量的节点。点击某个节点，展开微调表单（Hue 角度增减）。
- **中央舞台 (The Matrix Board) - 核心区：**
  - 一个巨大的网格系统。每一行是一个色系（如 Blue），每一列是一个亮度层级（如 500/50）。
  - **色块设计：** 每个深邃质感的色块上，展示当前的 Hex 值。右下角带一个小徽章，亮绿色的 AAA/AA 代表对比度通过，红色的 Fail 代表未通过。
  - **悬浮展开卡片 (Hover/Click State)：** 点击某个色块，下方（或内联展开）滑出专属仪表面板，提供 H、S、B 的输入框。数值拨动的同时，色块像水波一样平滑过渡。
- **右侧洞察面板 (Insight Chart)：**
  - **智能引导曲线：** 当你在中央舞台选中某一行色系时，这里渲染一张漂亮的折线图（X轴为阶梯 1-10，Y轴为 B 亮度或 S 饱和度）。
  - **安全绿洲 (Safe Zones)：** 图表背景里会有由于 W3C 对比度算法推导出的“绿色安全带”。线条若跌出了安全带区域，就意味着对比度挂了。这也是为什么你不需要漫无目的地拖拽，你可以看着图表直接微调 B 值，让折线回到“安全区”内。

### 2. 核心算法设计层 (Logic & Architecture)
为了实现你的想法，底层的代码逻辑需要这么设计：

- **W3C 算法反推：** 一般的做法是根据颜色正向计算对比度。但为了做到你说的“提供智能建议”，我们需要写一个逆向推导算法：给定前景色（黑/白），给定目标对比度（4.5:1），反得出一个亮度的阈值（Threshold），以此来画出右侧图表中的“安全区”。
- **颜色名称字典算法：** 构建一个 Hue (0-360) 映射表。例如 [345-15: Red], [15-45: Orange], [195-225: Blue]。每当色环角度改变，匹配函数自动分配 Name 并在前端渲染。
- **SVG 生成器 (Renderer)：** 将 React 状态里的颜色数据矩阵，按预定的间距（x, y 坐标定位）拼接成原生的 `<svg>` 节点字符串。在复制时写入操作系统的剪贴板，带有标准的 SVG MimeType，Figma 就能完美识别并粘贴为它的原生图层形式（Rects + Text Layers）。

### 3. 为什么这个方案很绝妙？
原先的曲线拖拽是结果导向的穷举，而你的设计是系统工程化的推导。

- **防呆设计：** 中分原则和对比度自动计算，让非专业的设计师或哪怕是开发人员，都不可能生成出“看不清字”的工业垃圾色块。
- **极速扩张：** 当需要支持一套 SaaS 系统的多主题时，只需拨动色相环预设为 24色，瞬间就能繁衍出 24 * 10 = 240 个全部符合 W3C 标准的可用色板，生产力爆表。
